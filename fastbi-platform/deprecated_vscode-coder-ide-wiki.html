Visual Studio Code - How to start.

1. Open the main page and click on the Coder icon. In the open window, select "Sign in with Fast.BI - <Customer Name>".

This will initiate the VSCode Cloud workspace IDE, your dedicated workspace for data modeling. Please wait until your workspace will be deployed and customized. It can take from 5-10min.


2. The first time you open your dedicated workspace, you should confirm the authorship of files. Check "Trust the authors of all files in the parent folder 'home'" and then click "Yes, I trust the authors."

3. On the Welcome screen, you can customize themes and other settings to tailor your workspace. When you're finished, click "Mark Done" at the bottom.


4. Now, just a few steps remain to start coding. For the first time, you need to create a new SSH key and login to Google Cloud SDK. Click on "three lines" options icon in the upper left corner, when the window pops up, select Terminal >> New Terminal.

At the bottom of the screen, a Terminal will open. Here, you can run all your commands. To generate a new SSH key, copy and paste the following command: ssh-keygen -t ed25519 -C "your_email@example.com" (don't forget to replace your email address). Then press the "Enter" key. The SSH key generation wizard will prompt you for information; you can generally press "Enter" to accept defaults until it's done generating the key.

Once the new key is generated, you need to copy the public key and add it to your Repo Account SSH keys. To retrieve the key, run the following command in the terminal: cat /home/coder/.ssh/id_ed25519.pub. Copy the output, go to "https://gitlab.fast.bi/-/profile/keys," click "Add new key," paste your previous output, and then click "Add key."

After successfully adding your new key, return to your workspace IDE and proceed to the next step (step 5).


5. Cloning your repository*. (Skip this step if you are working in Fast.BI-Coder, the repo already present in the IDE)
In the main page, click on the GitLab icon. It will open your code repository space. Navigate to the Data-Model folder and click on the DBT-Data-Model repository. In the upper-right corner, you will find a "Clone" icon. Click on it and copy the "Clone with SSH" repository link. Then, go back to the terminal in your workspace IDE and run the following command: git clone git@gitlab.fast.bi:bi-platform/terasky/data-model/dbt-data-model.git. If prompted to continue, type "yes."


6. With your data modeling repository and dbt projects cloned successfully, you now need to authenticate into the Google Cloud Platform. To do this, run a few commands.
First, execute the command: gcloud auth login. This will open a Google authentication web link. Follow the instructions to obtain an authorization code. Copy the provided authorization code, paste it into the terminal, and press "Enter" on your keyboard.
Next, run a similar command, but this one is necessary for proper dbt engine functionality: gcloud auth application-default login. Follow the same steps as before. After successfully logging into the Google Cloud Project, you can start working with your dbt projects.


7. The last step is to choose your DBT Project.
Click on "three lines" options icon in the upper left corner. Then choose File >> Open Folder...

On the TOP you'll see the "Open Folder" window pop-up, now you'll need to choose your dbt project folder. In our case we need to choose fast_bi_notebook >> dbt-data-model >> jaffle_shop_dbt_elt. Our dbt project is  jaffle_shop_dbt_elt. Then Click OK.

At first time using the Coder IDE you will get notification messages to connect to Turntable DBT extension, dtb Power user extension and GIT extension.

First for the Git (Extension), click on "Always".
Second for the dbt Power User (Extension) click on "Ignore".
Third for the Turntable for dbt Core (Extension) click on "Log In" and follow the instruction.

Then Extension window pops-up. Click on "Allow" button. 

Another External Website window pops-up. Click on "Open" button.

Now you was transfered to Turntable Extension Authenticate site. Click on "Continue with Google" button.

Choose your Google Fast.BI account.


After authentication is successfully finished, click on the "Get access code" button.

Copy the code, and back to Fast.BI Coder online IDE workspace. At the lower right corner, you see Signing into Turntable... Click on "Cancel" button, or if you see already Failed to sign in to Turntable click on "Enter Access Code" button. 

Paste your access code, and click enter.


8. Start your data modeling - in the left corner, at workspace ide, you will see your dbt project - dbt-data-model - files structure.
Open model directory, choose one of the model and right-click on it.
In the open window, select "Open in Integrated Terminal."

A new terminal window will open at the bottom of the screen, allowing you to interact with data-modeling model.

PROBLEMS - Here you will be informed about all issues related with your data model.

OUTPUT - The information windows about compilation process.

TERMINAL - Command line window.

LINEAGE - Your DBT Project lineage.

DOCUMENTATION EDITOR - DBT Project Model metadata documentation.

COLUMN-LEVEL LINEAGE - Track your transformation for columns.

QUERY PREVIEW - See your model output as data.

For guidance on working with all platform services during development, refer to the next page, "Fast.BI-DataModel-Development."
